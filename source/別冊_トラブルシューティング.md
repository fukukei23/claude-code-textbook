# 別冊 トラブルシューティング — よくあるエラーと直し方

> Claude Codeを使っていて遭遇するありがちなエラーと、その直し方。

---

## インストール関連

### 「claude: command not found」

```bash
# 解決法1: パスを通す
export PATH="$PATH:$(npm bin -g)"

# 解決法2: npmで再インストール
npm install -g @anthropic-ai/claude-code

# 解決法3: 公式インストーラー
curl -fsSL https://claude.ai/install.sh | bash
```

### WSL2 が動かない

```powershell
# PowerShellで実行
wsl --install
# PC再起動が必要
```

---

## 認証関連

### 「Authentication failed」

```bash
# 認証情報をリセット
claude auth logout
claude auth login
```

### APIキーが認識されない

```bash
# 環境変数に設定
export ANTHROPIC_API_KEY="sk-..."

# または ~/.claude/settings.local.json に記載
```

---

## 実行関連

### 「Permission denied」で実行できない

- Claude Codeが確認を求めている。`y` で許可して
- 危険なコマンドは手動で `y` を入力すること

### 「File not found」でファイルが見つからない

```bash
# 現在のディレクトリを確認
pwd

# ファイル是否存在を確認
ls ファイル名

# フルパスで指定
claude "Read 絶対パス/ファイル名"
```

### 日本語が文字化けする

```bash
# UTF-8設定を確認
export LANG=en_US.UTF-8
```

---

## パフォーマンス関連

### レスポンスが遅い / 止まる

| 原因 | 対策 |
|---|---|
| 長いセッション | `/compact` で圧縮 |
| MCPサーバーが多すぎる | 使わないサーバーを切る |
| ネットワーク遅延 | VPN/プロキシ確認 |

### コンテキスト上限に達した

```
/compact
```

で会話コストを削減。解決しない場合は `/clear` で新規セッション開始。

---

## Claude Code CLI固有

### 「WSL path」エラー

Windows側で実行している可能性。WSL2内から起動し直す。

```bash
# WSL2内から起動
claude
```

### 「Session locked」

別のセッションが実行中。完了を待つ、または終了させる。

## 覚えておきたいこと

```
┌──────────────────────────────────────┐
│  エラーが出たら：                        │
│                                      │
│  1. エラーメッセージを全文読む           │
│  2. 問題の一行前で「？」を入力してみる    │
│  3. `/help` で公式ドキュメント参照       │
│  4. それでも解決しない → 別セッションで   │
│     問題を再現して質問する               │
└──────────────────────────────────────┘
```
