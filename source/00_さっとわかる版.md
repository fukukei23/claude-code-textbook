## AIとは？

### 一言でいうと

AI（人工知能）は、大量のテキストを読んで「こういう時にはこう答える」を学習したコンピュータープログラムです。

| ポイント | 説明 |
|---|---|
| **文章を理解する** | 日本語の質問を理解して答えを生成する |
| **知識が豊富** | たくさんのデータを学習しているので幅広く答えられる |
| **嘘をつくことがある** | 自信を持って間違ったことを言うことがある（ハルシネーション） |

> ⚠️ **重要**: AIの答えを信じる前に、自分でも確かめよう！

### AIとClaude Codeの関係

- **AI**: 答えを考える部分（頭脳）
- **Claude Code**: AIを使ってコードを書いてくれる道具

---

## Claude Code 超まとめ

### 一言でいうと

Claude Codeは、**AIにコードを書かせる道具**です。コマンドを入力すると、AIが代わりにコードを書いてくれます。

### 使えるコマンド

| コマンド | 意味 |
|---|---|
| `claude` | 対話モードで起動 |
| `claude "質問"` | 1回だけ質問して終了 |
| `/help` | ヘルプを表示 |
| `/clear` | 会話をリセット |
| `/exit` | 終了 |

### 便利な機能

| 機能 | 役割 |
|---|---|
| **スキル** | よく使う指示を保存して自動化 |
| **Memory** | セッションをまたいで覚えておく |
| **フック** | 特定のタイミングで自動実行 |
| **MCP** | 外部サービスを接続して機能を拡張 |

## プロンプトのコツ

### 良いプロンプトの3原則

| 原則 | 例 |
|---|---|
| **具体的に** | 「ファイルを作成して」ではなく「todo-appフォルダにindex.htmlを作成して」 |
| **結果を指定** | 「表形式で出して」「日本語で答えて」 |
| **段階的に** | 複雑な作業はまず大枠を説明してから詳細を伝える |

### よく使うパターン

```
✅ 良い例: 「Pythonで hello.py を作成。中身は Hello World と表示する」
❌ 悪い例: 「作って」
```

## GitHub 超まとめ

### 一言でいうと

GitHubは、**コードをクラウドに保存できるサービス**です。消えない・変更履歴が残る・他の人と共同できます。

### 基本の流れ

```
作業 → git add（選択）→ git commit（記録）→ git push（アップロード）
```

| コマンド | 意味 |
|---|---|
| `git clone URL` | 初回にリポジトリをコピー |
| `git add ファイル` | コミット対象を選ぶ |
| `git commit -m "説明"` | 変更を記録 |
| `git push` | クラウドに上げる |
| `git pull` | 最新版を取得 |

## 安全に使うための約束

### 3つの約束

```
① APIキーは外部に見せない
   → 環境変数で管理、.gitignoreに追加

② 危険な操作は確認を読んでから
   → 「本当に消しますか？」にはよく考えよう

③ 小まめにコミット
   → 問題が起きたら巻き戻せる
```

### APIキー流出の怖さ

```
❌ GitHubにAPIキーをpush → 世界中に公開 → 多額の請求が来る
❌ スクリーンショットにAPIキーを映す → SNSで拡散される
```

### お金の知識

- AIは「文字量」で料金が決まる
- `/context` で今のコストを確認できる
- 長いセッションは費用が高額になる

## 知識管理（SSOT）の概念

### SSOT = Single Source of Truth

「情報が散らばるとややこしいので、**正しい情報を1箇所にまとめる**」という考え方。

```
❌ 困ったとき
   - メモ帳Aには○○と書いてある
   - メモ帳Bには××と書いてある
   - どちらが正しいか分からない！

✅ SSOTなら
   - 正しい答えはここにしかない
   - 迷ったらここを見ればいい
```

### 日記の書き方

```
## セッションログ (時刻)
- やったこと（3〜5行）
- 詳細: ファイル名
```
