Claude Code を始めよう

この章では、Windows + WSL2 環境に Claude Code をインストールする方法を説明します。Linux ユーザーは同じ手順でインストールできます。

必要なもの

項目 最小要件
OS Windows 10 (version 2004以上) または Windows 11
WSL2 Ubuntu 20.04 以降
Node.js バージョン 18 以上
メモリ 4GB 以上（8GB 以上推奨）

WSL2 とは？ Windows Subsystem for Linux 2 の略。Windows上でLinuxを動かせる仕組みです。

Step 1: WSL2 をインストールする

PowerShell を管理者権限で開く

  1. Windows キーを押す
  2. 「PowerShell」と検索
  3. 「管理者として実行」をクリック

WSL2 を有効化する

# WSL と仮想マシンプラットフォームを有効化
wsl --install

# 再起動を促されたら再起動する

初期設定

再起動後、Ubuntu が起動し、ユーザー名とパスワードの設定を求められます。

Installing, this may take a few minutes...
Please create a default UNIX user account.
New password: ********
Confirm password: ********

Step 2: Node.js をインストールする

# パッケージリストを更新
sudo apt update && sudo apt upgrade -y

# Node.js 18.x をインストール
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# バージョン確認
node --version    # → v18.x.x と表示されればOK
npm --version     # → 9.x.x と表示されればOK

Step 3: Claude Code をインストールする

推奨: 公式インストーラー（自動更新対応）

# Linux/macOS/WSL2 の場合
curl -fsSL https://claude.ai/install.sh | bash

代替: npm でインストールする

npm install -g @anthropic-ai/claude-code

# 確認
claude --version

Step 4: 認証する

Claude Code を初めて起動する

claude

方法A: OAuth（推奨）

初回起動時、ブラウザで Anthropic アカウント認証を求められます。

  1. 表示される URL をクリック（またはコピーしてブラウザで開く）
  2. Anthropic アカウントでログイン
  3. 許可ボタンをクリック
  4. ターミナルに戻る

方法B: API キー

# 環境変数に設定
export ANTHROPIC_API_KEY=your-api-key-here

# 永続化する場合
echo 'export ANTHROPIC_API_KEY=your-api-key-here' >> ~/.bashrc
source ~/.bashrc

API キーを取得: Anthropic Console でアカウントを作成し、API Keys から生成できます。

Step 5: WSL2 固有の設定

Windows + WSL2 環境で使う場合、ブラウザ連携を設定します。

Windows ブラウザを設定する

# ~/.bashrc に追加
echo 'export BROWSER="/mnt/c/Program Files/Google/Chrome/Application/chrome.exe"' >> ~/.bashrc
source ~/.bashrc

他のブラウザを使う場合のパス:

ブラウザ パス
Chrome /mnt/c/Program Files/Google/Chrome/Application/chrome.exe
Edge /mnt/c/Program Files (x86)/Microsoft/Edge/Application/msedge.exe
Firefox /mnt/c/Program Files/Mozilla Firefox/firefox.exe

Step 6: 設定ファイルを作成する

Claude Code は ~/.claude/ ディレクトリ内のファイルで設定を管理します。

settings.json の作成

# 設定ディレクトリを作成
mkdir -p ~/.claude

# 設定ファイルを作成
cat > ~/.claude/settings.json << 'EOF'
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(git *)"
    ],
    "deny": [
      "Bash(curl *)",
      "Read(./.env)"
    ]
  }
}
EOF

settings.json の役割: Claude Code に許可する操作（コマンド実行、ファイル読み書きなど）を指定します。

CLAUDE.md の作成（プロジェクト固有ルール）

プロジェクトフォルダに CLAUDE.md を作成すると、プロジェクトのルールを Claude Code に知らせられます。

# プロジェクトフォルダに移動
cd ~/your-project

# CLAUDE.md を作成
cat > CLAUDE.md << 'EOF'
# プロジェクトルール

## コーディング規約
- 日本語コメントを使用
- インデントはスペース2つ

## 一般的なタスク
- 新しいファイル作成時は必ずテストを書く
- コミット前にリントを実行
EOF

Step 7: 動作確認

# バージョン確認
claude --version

# ヘルプ表示
claude --help

# 対話モードで起動
claude

初回テストとして、以下を入力してみましょう:

say: 「こんにちは！元気？」

応答があれば、正常に動作しています。

トラブルシューティング

wsl: command not found

原因: WSL2 が有効化されていない

解決:

# PowerShell（管理者）で実行
wsl --install
# PCを再起動

claude: command not found

原因: インストール失敗、または PATH が通っていない

解決:

# npm のグローバルパス確認
npm bin -g

# PATH に追加
echo 'export PATH="$(npm bin -g):$PATH"' >> ~/.bashrc
source ~/.bashrc

Permission denied（npm インストール時）

解決:

# npm のパーミッション設定を変更
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

API key not found

原因: 環境変数 ANTHROPIC_API_KEY が設定されていない

解決:

export ANTHROPIC_API_KEY=your-api-key-here
echo 'export ANTHROPIC_API_KEY=your-api-key-here' >> ~/.bashrc

OAuth 認証が失敗する

解決:

  1. ターミナルに表示される URL を手動でブラウザで開く
  2. 認証を完了する
  3. ターミナルに戻る
